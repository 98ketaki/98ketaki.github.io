Machine Learning

Multi-omic Lung Cancer Analysis
Problem: subtype classification depends on high-dimensional molecular signals across multiple omics layers.
Built: a PyTorch autoencoder for unsupervised feature selection on TCGA gene-expression, CNV, and clinical data, with PCA, t-SNE, SVM & Random Forest comparisons.
Result: the CNV + gene-expression ensemble outperformed mono-omic approaches — showing the value of integrating multi-omic data.

Machine Learning

Malaria Detector
Problem: manual blood-smear screening is slow and uneven in low-resource settings.
Built: a ResNet50 transfer-learning classifier (PyTorch / timm) for infected vs. healthy cells, with augmentation.
Result: fine-tuned over 10 epochs (batch 32, Adam, LR scheduling) and evaluated on held-out smears — code & report in the repo.

Bioinformatics

Variant Calling Pipeline
Problem: reproducibly finding SNPs & indels from raw paired-end reads.
Built: an end-to-end GATK pipeline (BWA → samtools → Picard → HaplotypeCaller) on an AWS EC2 + conda setup.
Result: produced hard-filtered, separated SNP & indel tables ready for downstream analysis and IGV visualization.

Bioinformatics · ML

RBP Peak Finder
Problem: understanding viral infection requires finding where host RNA-binding proteins interact with viral RNA.
Built: a two-part pipeline using eCLIP data to identify mapped-read peaks, then a CNN to distinguish binding peaks from background.
Result: a workflow to predict whether a host protein binds viral RNA and highlight likely binding regions.
